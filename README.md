# Data Integrity Automation Prototype

## Overview

This project is a mock fleet management business excellence prototype. It is designed to show how messy operational data can be profiled, cleaned, validated, automated, and visualized using tools such as SQL, Alteryx, Power Automate, Power Apps, and Tableau.

The project focuses on data integrity, reproducibility, and practical business insight.

## Main Problem

The prototype simulates a situation where fleet-related data is spread across multiple sources and contains inconsistencies. These issues make it difficult to trust dashboards, reports, and operational decisions.

## Tools Used

- SQL Server / SQL Server Management Studio
- Alteryx
- Power Automate
- Power Apps
- Tableau
- Markdown documentation
- Git / Github

## Planned Workflow

1. Create or collect mock fleet-related datasets
2. Profile the raw data
3. Identify data quality issues
4. Clean and standardize the data
5. Load raw and cleaned data into SQL
6. Create data quality exception outputs
7. Build a simple automation flow
8. Build a simple exception-review app
9. Create a dashboard for business insights
10. Document the process so it is reproducible

## Project Purpose

This project is intended to demonstrate initiative, fast learning, analytical thinking, and a structured approach to messt business data. It is not intended to represent production-level expertise in every tool used.
